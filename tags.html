Experimental

These are features that are currently in their primal stage. This include features that are not ready for production use and may change a lot during versions. These features have a chance to be added or not to further versions.

Features marked as experimental are unstable, and may cause bugs and errors to occur. They are included within versions for the purpose of being tested and to receive feedback.
It is recommended to not use experimental features in any non demo project.

To enable experimetal features

For js-plugins, you can enable experimental features by setting [plugin class].experimental to true. If the plugin has more than one class, their toggle will be on their global configuration class. Which is the same to enabling experimental features on plugins with one class

For others, the steps will be on their manual.


Deprecated

These features are no longer recomemended to be used. These might be removed or replaced on future versions. They may only be kept for compatibilty reasons.

If you are planning to only use one version of a product or program, you don't have to worry about deprecated features being removed.


Removed features within documentation

Documentation of removed experimental features will be gone. They may only return once the feature's name also does.

For removed deprecated features. Their documentation will stay but separated from their main page, due to the feature being recommended from previous versions.
They will still be accessible on their scripting reference's main page's menu through Experimental and Deprecated Features.
