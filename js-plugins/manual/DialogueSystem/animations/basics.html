Basics

A main part in making animations are DialogueAnimation objects. They are used to store animation data.
Within the object, there are properties that are used within animations.

animation - string where the CSS animation property is stated, but you cannot state it's duration here.
duration - number that sets the duration of the animation in seconds.
wait - when true, the animation will be waited to finish before moving to the next tasks.
callback - function which will be called when the animation finishes.


To actually set animations

You've got to use a DialogueAnimationConfig. Every manager has an animation config as it's animation property. This is where animations are stored for transitions.
It has these as properties.

box - transitions for the dialogue box.
portrait - transitions for the portrait image.
lineClear - a DialogueAnimation that is played to the text line when it clears.
The box and portrait animations are stored by DialogueBoxNPortraitAnimationData objects, which have 2 DialogueAnimation properties.
onEnable - played when those objects enable.
onDisable - played when those objects disable.
