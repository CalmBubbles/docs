Controlling

The system has several variables you can use to control time flow.

targetFrameRate

This is the target frames per second (FPS) of the system. How frequently frame updates in one second. This variable controls how the whole system runs.

The higher the value, the more frequent updates happen and feel smoother. But the higher the value, the more power it requires to run. If you set it on a high value and use it on a device that cannot handle it. Rather than making the system feel smoother, you're gonna make it feel worse.

By default this is 60 FPS, which is recommended.

timeScale

This value controls how fast time seems to flow. Scales deltaTime and affects time. The higher the value, the faster the time. This does not affect FPS and how frequent updates happen.

The default value is 1.

maximumDeltaTime

Limits the value of deltaTime before passing it to timeScale. When lag occurs deltaTime increases a lot, which may show unexpected results. This value is the maximum allowed value of deltaTime.

By default, 0.1111111.
