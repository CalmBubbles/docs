File paths

File paths are used in loading resources. They can get pretty long, it's quite fine if you're loading only a few resources. But if you're loading bunch of resources, things start to get confusing.

But if you're using resource arrays, you can lessen confusion. In the constructor, pass the resources' directory path as a second parameter. This will be applied to all resources in that array. 

JavaScript
You can set the array's directory to a text pointing to the location, or use DialogueDirectoryType.

By default, resource arrays have their directory path as DialogueDirectoryType.Root or the page's root location.

You can also set the directory to the current page's location with DialogueDirectoryType.Same. This will make the resources have their file paths relative to the current page.
