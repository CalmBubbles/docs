Resource arrays

Loading bunch of resources with just their class constructor, especially with long file paths, is a headache and is confusing.

Instead of using the default classes, use their array variants. Easily scalable. For sprites, use DialogueSpriteArray. For audio clips, use DialogueAudioArray. 

JavaScriptdiaManager.config.sprites = new DialogueSpriteArray(
    [
        {
            name: "proto", // name
            src: "proto1.png" // source file
        },
        { name: "protoc", src: "proto1c.png" },
        { name: "proto_damn", src: "proto_jealous.png" },
        { name: "proto_stare", src: "proto_gasp2.png" },
        { name: "proto_e", src: "proto_gasp1.png" },
        { name: "rue", src: "rue.png" },
        { name: "rue_oh", src: "rue_oh.png" },
        { name: "rue_talk", src: "rue_talk.png" }
    ],

    // file path/directory
    "img/Faces"
);

diaManager.config.audios = new DialogueAudioArray(
    [
        { name: "default", src: "text.wav" },
        { name: "robot", src: "text_robot.wav" }
    ],
    "audio"
);

By default, the file paths of the resources are relative to the page's root location.
