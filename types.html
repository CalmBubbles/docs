Integer

Integers are whole numbers. They can be positive or negative.

Integer types strictly exist in most programming languages. But in dynamic languages like JavaScript, they exist, but not strictly.

In JavaScript, integers are just numbers.
It's just our fancy way of saying "this is a whole number".

Examples of integers

  • -1
  • 420
  • 9 + 10
  • 3e8

Examples of non integers

  • -2.2
  • 1 + 0.002
  • 1e-10
  • 1.0 (only on strictly typed languages)

Why state something as an integer?

We ain't stopping you from using 0.1 on something indicated as an integer. But behaviors may break if you pass on decimal values to those.

We recommend you to use whole numbers on things marked as an integer.


Void

In JavaScript, methods can return values. Normally they return undefined. But documenting it with the word undefined sounds wrong and broken. Instead, we use the word void like strictly typed languages.

It has the same meaning of "nothing". But unlike undefined which sounds like we forgor to make it return something, void sounds like it's supposed to return nothing. And it's way cooler.


Any

Since JavaScript is a dynamically typed language, we can give any variable and property a value of anything.

But we normally don't want that, so we document the actual types we want you to use. But just in case, we will still document property types that we don't give a care about, using the word any.
